Core CS Concepts

This page assumes you know nothing about programming or computer science (CS) and lays a foundation you can build your knowledge upon.

What is a computer?

A computer is made up of:

  1. A mother board (Main circuit board that other circuit boards/drives are plugged into)
  2. A CPU (Collection of processors that perform certain functions, given certain code)
  3. RAM (Short-term memory)
  4. Storage Device (Long-term memory)
  5. Graphics card (Like a CPU, but used for graphical rendering onto a monitor)

A computer is designed to take a set of instructions (code) and perform tasks outlined in those intructions.

What is a server?

A server is a type of computer, made of the same components listed above.

In most households, we have Personal Computers. These are designed with sufficient resources (storage, memory, processing power) to perform tasks that everyday individuals need to do, as well as be cost-efficient so every household can afford one.

Servers are much more powerful (and expensive) computers, that are designed for more complex tasks, like hosting a social media application.

What is "The Cloud"?

"The Cloud" is a term given to a collections of one or more servers that are typically accessible to people. There are big buildings all over the world that each contain many servers. When you "upload a picture to the cloud", you are actually sending your picture to one of these servers to store.

How does a computer read code?

A computer, more specifically a processor (CPU), is designed to read something called "Machine Code". Machine code is impossible for humans to write, so we've developed "higher-level" languages, like JavaScript, Python, C++, etc. These languages are much easier to write, but they need to be translated into Machine Code before a computer can understand it. This is done with an "Interpreter" and/or a "Compiler"

What are Interpreters and/or Compilers?

Compilers are little programs that take code you write and translate it into machine code. Machine code is "archiceture specific". This means if you compile your code on one computer, it will only be able to run on similar computers. For example: Code that is compiled on a Mac, can not run on a PC.

The flow would be something like this: You write some code to a file, you run a compiler and give it that files, it compiles it into Machine Code and saves it to another file called an "Executable". You can then run your executable just like any other program.

Interpreters are similar, in that they translate the code you write into Machine Code, however they work a little differently. They translate the code and execute it as your program runs.

The flow would be something like this: You write some code to a file, you run the interpreter and give it your file, it translates the file and runs the Machine Code through the processor in realtime.

Examples of compiled languages are C, C++, Go, Rust.
Examples of interpreted languages are Python, JavaScript, Ruby.

Note: Interpretted languages are generally "slower" to run than compiled, because the process of translation is longer, however interpretted languages are typically easier to learn and use and therefore quicker to use in development.
